# How to Use This Garden

## Writing notes

1. **Create a new `.md` file** in the `content/` folder (or a subfolder)
2. **Write freely** — you don't need a perfect essay, just your raw thinking
3. **Link as you go** — when you mention a concept, wrap it: `[[Opportunity Cost]]`
4. **Commit and push** — GitHub Actions builds and publishes the site automatically

## The linking mechanic

When you type `[[Note Name]]` inside any note, Quartz creates a clickable link to that note. If the note doesn't exist yet, it becomes a placeholder — a reminder to expand that idea later.

**This is the core habit:**
Every time you write about something, ask: *"What existing idea does this connect to?"* Then link it.

Over time, the graph view reveals the architecture of your thinking.

## Folder structure

```
content/
├── index.md          ← Home page
├── Start Here.md     ← This file
├── Concepts/         ← Evergreen ideas (principles, frameworks)
├── Projects/         ← Notes tied to specific projects
├── Reflections/      ← Daily/weekly reflections
└── Templates/        ← Reusable note templates
```

## Daily workflow

1. Open a new `Reflections/YYYY-MM-DD.md` file
2. Use the [[Templates/Daily Reflection]] template
3. Write what you're thinking about
4. Add `[[links]]` to existing concepts or create new ones
5. Push to GitHub → live in ~2 minutes

## The "ah-ha" mechanic

The magic happens when you start seeing the same underlying pattern in different domains.

**Example from [[Concepts/Opportunity Cost]]:**
A concept from economics applies directly to how you should prioritize projects — and also explains why giving up on one path to pursue another is rational, not failure.

When you link ideas across domains, the graph lights up, and so does your thinking.
